# Frontend Application

This directory contains the implemented Next.js presentation layer for the churn prediction repository.

## Purpose

The frontend consumes analytics artifacts generated by [`main()`](../src/run_pipeline.py:110) and presents them through route-based business pages.

Current pages:
- dashboard via [`HomePage()`](src/app/page.tsx:12)
- model comparison via [`ModelsPage()`](src/app/models/page.tsx:4)
- retention priority via [`RetentionPage()`](src/app/retention/page.tsx:4)
- explainability via [`ExplainabilityPage()`](src/app/explainability/page.tsx:4)
- business insights via [`InsightsPage()`](src/app/insights/page.tsx:4)

## Data Contract

The frontend reads repository outputs through [`frontend/src/lib/data.ts`](src/lib/data.ts), including:
- [`outputs/tables/dataset_overview.csv`](../outputs/tables/dataset_overview.csv)
- [`outputs/tables/model_performance_comparison.csv`](../outputs/tables/model_performance_comparison.csv)
- [`outputs/tables/customer_priority_table.csv`](../outputs/tables/customer_priority_table.csv)
- [`outputs/tables/top_20_retention_targets.csv`](../outputs/tables/top_20_retention_targets.csv)
- [`outputs/tables/logistic_coefficients.csv`](../outputs/tables/logistic_coefficients.csv)
- figures under [`outputs/figures/`](../outputs/figures/)

PNG figures are served to the browser through [`GET()`](src/app/api/assets/[fileName]/route.ts:7).

## Development

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm run dev
```

Open `http://localhost:3000` in the browser.

## Quality Checks

Run lint:

```bash
npm run lint
```

Run production build:

```bash
npm run build
```

## Implementation Notes

- The frontend uses Next.js App Router.
- The frontend remains isolated from the Conda-managed Python environment.
- The analytics pipeline should still be rerun through [`src/run_pipeline.py`](../src/run_pipeline.py) when output artifacts need to be refreshed.
